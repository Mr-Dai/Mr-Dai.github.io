Preface

This is a cookbook for Subversion CLI usage. No useless sentence should be found here.

Let's begin.

Check out a working copy

Everything starts from getting your own working copy. Like checking out books from library, we use svn checkout to check out from repository. 

$ svn checkout http://svn.example.com:9834/repos

Othen than http scheme, file:// is also supported: 

# On Unix/Linux
$ svn checkout file:///var/svn/repos
$ svn checkout file://localhost/var/svn/repos
# these two are the same

# On Windows
C:\> svn checkout file:///X:/var/svn/repos
C:\> svn checkout "file:///X|/var/svn/repos"

Note how we use forward slashes to represent file path in Windows, even though the native form should use backslashed. As for the second expression, the URL should be quoted so that the vertical bar wouldn't be interpreted as a pipe.

The list is not exhaustive yet. The URL scheme supported by svn checkout can be listed as below:

Schema Access method
file:/// Direct repository access (on local disk).
http:// Access via WebDAV protocol to Subversion-aware Apache server.
https:// Same as http://, but with SSL encryption.
svn:// Access via custom protocol to an svnserve server.
svn+ssh:// Same as svn://, but through an SSH tunnel.
